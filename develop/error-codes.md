# <a name="partner-center-rest-error-codes"></a>Partnerközpont – REST hibakódok

**A következőkre vonatkozik:**

- Partnerközpont
- A 21Vianet által üzemeltetett partneri központ
- A Microsoft Cloud Germany Partnerközpontja
- A Microsoft Cloud for US Government Partnerközpontja

A partner Center REST API-k egy állapotkódot tartalmazó JSON-objektumot adnak vissza. Ez a kód jelzi, hogy a kérelem sikeres volt-e, vagy hogy miért sikertelen volt.

## <a name="success-responses"></a>Sikeres válaszok

A **2xx** azt jelzi, hogy az ügyfél kérelme sikeresen megérkezett, megértette és elfogadták.

## <a name="http-status-codes"></a>HTTP-állapotkódok

A következő **4xx** és **5xx** -állapotkódok hibát jeleznek:

- 400: hibás kérelem
- 401: nem engedélyezett
- 403: tiltott
- 404: nem található
- 405: a metódus nem engedélyezett
- 406: nem elfogadható
- 409: ütközés, hibakód
- 412: nem sikerült az előfeltétel
- 429: túl sok kérés
- 500: belső kiszolgálóhiba
- 501: nincs implementálva
- 502: hibás átjáró
- 503: a szolgáltatás nem érhető el
- 504: átjáró időtúllépése

## <a name="error-responses"></a>Hibaüzenetek

A **4xx** -vagy **5xx** -állapotkód bármilyen válasza tartalmaz egy hibaüzenetet, amely további részleteket tartalmaz az észlelt hiba állapotáról.

A következő táblázat és kód minta egy adott hiba sémáját ismerteti:

| Név        | Típus   | Leírás                                                                                                                                            |
|-------------|--------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| code        | sztring | Mindig vissza kell adni. Azt jelzi, hogy milyen típusú hiba történt. Nem null értékű.                                                                                  |
| leírás | sztring | Mindig vissza kell adni. Részletesen leírja a hibát, és további hibakeresési információkat biztosít. Nem null, nem üres. A maximális hossz 1024 karakter. |
| adatok        | array  | Csak bizonyos típusú hibák esetén tért vissza. A hibás objektumok listája.                                                                                           |
| source      | sztring | Mindig vissza kell adni. A hiba forrása.                                                                                                              |

```json
{
  "code": <string>,
  "description": <string>,
  "data": [

  ],
  "source": <string>
}
WWW-Authenticate: OAuth realm=urn:cpsvc:cpid:{some cid}
```
